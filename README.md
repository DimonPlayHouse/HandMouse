
1. Введение
 
1.1 Наименование проекта
«Разработка системы управления курсором на основе движений рук» (рабочее название: HandMouse).

1.2 Цель проекта
Разработка приложения на языке C++, которое с помощью обычной веб-камеры отслеживает положение и жесты руки пользователя в реальном времени и преобразует их в движения курсора мыши и клики. Приложение должно использовать нейросетевые методы для точного определения ключевых точек кисти (скелет руки).

2. Основание для разработки
Проект выполняется в рамках проектной работы.

3. Назначение разработки
Программа предназначена для бесконтактного управления компьютером, что может быть полезно для людей с ограниченными возможностями, в образовательных целях, для использования в больницах для обеспечения стерильности.

4. Требования к программе
   
4.1 Функциональные требования
Захват видео с веб-камеры (встроенной или внешней) в реальном времени.

Обнаружение руки в кадре с использованием нейросетевой модели TensorFlow Lite.

Определение ключевых точек кисти.

Трекинг руки во времени с сохранением идентификатора руки (поддержка одной руки).

Управление курсором:

Положение курсора на экране должно соответствовать положению указательного пальца в пространстве, спроецированному на плоскость экрана.

Чувствительность перемещения должна настраиваться.

Распознавание жестов для кликов:

Левый клик: например, соединение большого и указательного пальцев («щелчок») или удержание пальцев вместе в течение короткого времени.

Правый клик: другой жест, например, соединение большого и среднего пальцев.

Двойной клик: двукратное быстрое соединение пальцев.

Жест «захват»: зажатие пальцев и перемещение руки.

Настройка параметров через простое меню: чувствительность, пороги срабатывания.

Визуализация: вывод обработанного видео с наложением скелета руки и индикацией текущего жеста для отладки.

4.2 Нефункциональные требования
Производительность:

Частота обработки кадров не менее 15–20 FPS на типичном оборудовании (процессор не ниже Intel Core i5, без дискретной графики).

Задержка от момента движения руки до перемещения курсора не должна превышать 100 мс.

Надёжность: программа не должна завершаться аварийно при потере камеры или временном отсутствии руки.

Простота использования: минимальное взаимодействие с пользователем (достаточно запустить приложение, рука появилась – курсор управляется).

4.3 Требования к интерфейсу
Графический интерфейс (опционально): окно предпросмотра с настройками (можно отключить для фоновой работы).

5. Состав и структура программы
   
5.1 Основные модули
Модуль захвата видео – на основе OpenCV.

Модуль детекции и трекинга руки – интеграция предобученной нейросетевой модели TensorFlow Lite.

Модуль интерпретации жестов – анализ ключевых точек и определение команд.

Модуль управления курсором – эмуляция мыши (перемещение, клики) с использованием системных API (Windows API).

Модуль визуализации – отображение видео и скелета.

5.2 Потоки данных

Основной цикл: получение кадра -> обработка нейросетью -> определение жестов -> эмуляция действий.

6. Используемые технологии и библиотеки
Язык программирования: C++14/17.

Система сборки: Conan.

Библиотеки:

OpenCV (последняя стабильная версия) – для захвата видео и базовой обработки изображений.

TensorFlow Lite – для инференса нейросетевой модели (если не используется готовая библиотека с C++ API).

MediaPipe (C++ API) – возможно использование готового решения для трекинга рук.

Для эмуляции мыши:

Windows: SendInput, SetCursorPos, mouse_event.

7. Требования к нейросетевой модели
8. 
Модель должна определять 21 ключевую точку кисти с высокой точностью.

Возможные варианты:

MediaPipe Hands (от Google) – легковесная, есть предобученная модель и C++ API.

Модели из OpenPose (Hand keypoints).

Модель должна работать в реальном времени на CPU, возможно использование GPU для ускорения.

8. Варианты использования 
Пользователь запускает приложение – автоматически включается камера, начинается трекинг. Курсор перемещается движением руки.

Пользователь выполняет жест «щелчок» – происходит клик левой кнопкой мыши.

Пользователь меняет чувствительность – через конфиг или GUI.

Пользователь временно отключает управление – нажатием горячей клавиши.

Приложение теряет руку из кадра – курсор остаётся на месте, клики не происходят до появления руки.

9. Этапы разработки
Анализ и проектирование – выбор библиотек, архитектуры.

Прототипирование – быстрая реализация захвата видео и вывода скелета на экран (визуализация).

Разработка модуля управления курсором – привязка координат руки к экрану, эмуляция движений.

Разработка модуля жестов – определение кликов по ключевым точкам.

Интеграция и отладка – настройка чувствительности, устранение задержек.

Создание интерфейса настроек.

Тестирование – на разных конфигурациях ПК, при разном освещении.

Составление документации.

10. Критерии приемки
Программа корректно захватывает видео с камеры.

При появлении руки в кадре через 1-2 секунды начинается управление курсором.

Движения руки плавно перемещают курсор по экрану.

Жесты кликов распознаются с точностью не менее 90% при нормальном освещении.

Программа не приводит к зависаниям и потребляет не более 30% CPU на среднестатистическом процессоре (без GPU).

Возможность настройки параметров без перекомпиляции.
