# Введение

## 1.1 Наименование проекта
«Разработка системы управления курсором на основе движений рук»  
(рабочее название: HandMouse)

---

## 1.2 Цель проекта

Разработать приложение на языке C++, которое с использованием веб-камеры отслеживает положение и жесты руки пользователя в реальном времени и преобразует их в управление курсором мыши.

Система должна применять нейросетевые методы для определения ключевых точек кисти (скелета руки) и обеспечивать точное и плавное управление курсором.

---

## 1.3 Основание для разработки

Проект выполняется в рамках учебной проектной работы.

---

## 1.4 Назначение разработки

Программа предназначена для бесконтактного управления компьютером.

Основные области применения:
- помощь людям с ограниченными возможностями
- использование в образовательных целях
- применение в медицинских учреждениях (соблюдение стерильности)
- альтернативный способ взаимодействия с компьютером

---

# 2. Требования к программе

## 2.1 Функциональные требования

### Работа с видео
- Захват видео с веб-камеры (встроенной или внешней) в реальном времени
- Обработка видеопотока без значительных задержек

### Обнаружение и трекинг руки
- Обнаружение руки с использованием нейросетевой модели (TensorFlow Lite или аналог)
- Определение 21 ключевой точки кисти
- Отслеживание положения руки во времени
- Поддержка одной активной руки

### Управление курсором
- Положение курсора соответствует положению указательного пальца
- Проекция координат руки на экран
- Настраиваемая чувствительность движения

### Распознавание жестов
Поддержка базовых жестов:
- Левый клик — соединение большого и указательного пальцев
- Правый клик — соединение большого и среднего пальцев
- Двойной клик — быстрое повторение жеста
- Перетаскивание — удержание жеста «захват» с движением руки

### Настройки
- Изменение чувствительности
- Настройка порогов распознавания жестов
- Возможность конфигурации через меню или файл

### Визуализация
- Отображение видеопотока
- Наложение скелета руки
- Индикация распознанных жестов (для отладки)

---

## 2.2 Нефункциональные требования

### Производительность
- Частота обработки: не менее 15–20 FPS
- Задержка управления: не более 100 мс
- Работа на CPU (уровень Intel Core i5 без GPU)

### Надёжность
- Отсутствие аварийных завершений при:
  - потере камеры
  - исчезновении руки из кадра
- Корректное восстановление работы

### Удобство использования
- Минимальное взаимодействие с пользователем
- Работа «из коробки» после запуска

---

## 2.3 Требования к интерфейсу

- Окно предпросмотра с видеопотоком (опционально)
- Возможность отключения визуализации
- Простое меню настроек или конфигурационный файл
- Поддержка горячих клавиш (например, включение/выключение управления)

---

# 3. Состав и структура программы

## 3.1 Основные модули

### Модуль захвата видео
- Реализуется с использованием OpenCV
- Получает кадры с камеры

### Модуль детекции и трекинга руки
- Интеграция нейросетевой модели (TensorFlow Lite / MediaPipe)
- Определение ключевых точек кисти

### Модуль интерпретации жестов
- Анализ положения точек
- Распознавание жестов пользователя

### Модуль управления курсором
- Эмуляция действий мыши
- Использование системных API (например, Windows API)

### Модуль визуализации
- Отображение видеопотока
- Отрисовка скелета руки и состояния системы

---

## 3.2 Потоки данных

Основной цикл работы приложения:

1. Захват кадра с камеры  
2. Обработка кадра нейросетью  
3. Определение ключевых точек  
4. Распознавание жеста  
5. Выполнение действия (перемещение курсора / клик)  

---

# 4. Используемые технологии

## 4.1 Язык программирования
- C++ (стандарт C++14/17)

## 4.2 Система сборки
- Conan (менеджер зависимостей)
- CMake

## 4.3 Библиотеки

- OpenCV — захват видео и обработка изображений  
- TensorFlow Lite — инференс нейросети  
- MediaPipe — готовые решения для трекинга рук (опционально)

## 4.4 Управление мышью

Для Windows:
- `SendInput`
- `SetCursorPos`
- `mouse_event`

---

# 5. Требования к нейросетевой модели

- Определение 21 ключевой точки кисти
- Работа в реальном времени
- Поддержка CPU (обязательно)
- Возможность ускорения через GPU (опционально)

### Возможные решения:
- MediaPipe Hands
- OpenPose (модуль рук)

---

# 6. Сценарии использования

- Пользователь запускает приложение — включается камера
- Система начинает отслеживание руки
- Движение руки управляет курсором
- Выполнение жеста — вызывает соответствующее действие:
  - клик
  - двойной клик
  - перетаскивание
- Пользователь может:
  - изменить чувствительность
  - временно отключить управление (горячая клавиша)
- При потере руки:
  - курсор фиксируется
  - действия не выполняются

9. Подготовка документации  

---

# 8. Критерии приемки

- Корректный захват видео
- Управление курсором начинается через 1–2 секунды после появления руки
- Плавное движение курсора
- Точность распознавания жестов не менее 90%
- Отсутствие зависаний
- Загрузка CPU не более 30% (без GPU)
- Возможность настройки без перекомпиляции
